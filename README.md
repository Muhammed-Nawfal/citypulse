# 🌆 CityPulse

### Type a city and a risk scenario. Watch an AI agent research live data and **build a 3D risk map of that city in real time** — zone by zone — as it thinks.

**Generative UI, on stage.** Not a chatbot with a pretty sidebar. The 3D scene *is* the agent's output: every node, every colour, every card is generated at runtime — and the agent even **chooses which kind of 3D scene to render** for your question.

## 90-second demo

1. **Type:** `London — flooding risk`
2. The agent researches live web data via **Linkup**. Progress streams into the sidebar as it thinks.
3. A 3D city grid **materialises zone by zone** over **AG-UI** as each research node completes — staged, deliberate, dramatic. High-risk zones pulse **red**, moderate **amber**, safe **green**.
4. **Click any zone** → the agent generates an **A2UI** detail card: risk level, evidence bullets, Linkup source citations, timestamp.
5. **Ask:** `What if we add a flood barrier at zone 4?` → the agent re-evaluates, re-colours the affected zones, and emits an impact-assessment card.
6. **Switch scenario:** `Now show me UK power grid resilience` → the scene **dissolves and re-assembles as a completely different 3D interface** — a network graph of power stations and transmission lines. Same input box. The agent *chose* what to render.

> Every zone you see was researched from live web data seconds ago. Every card was generated by the agent. The UI didn't exist before the query. **That's generative UI** — not a dashboard, not a chatbot. An agent that builds its own interface.

## Why CityPulse wins — mapped to the judging criteria

### 🎨 Most creative use of Generative UI + A2UI
The headline mechanic is the **Dynamic Scene System**: the agent doesn't fill a fixed template — it emits a **scene blueprint** and *decides the entire 3D interface*. `"London flooding"` builds an isometric **city grid**; `"UK power grid resilience"` builds a **network graph** of nodes and connections. Two completely different generative interfaces from the same input box, both driven by agent-authored A2UI. On top of that, every zone detail and impact panel is an **A2UI v0.9 surface** the agent composes at runtime — judges can watch the raw envelopes fire via the inline mirror pill.

### 🔌 Integration of CopilotKit, AG-UI, and A2UI
The full CopilotKit stack carries the experience end-to-end: the **LangGraph** agent emits A2UI envelopes, **AG-UI** streams them as the agent reasons, **CopilotKit** wires them through Next.js, and the **A2UI renderer** turns them into live React — the 3D scene and the generated cards alike. **Linkup** grounds every score in live web data and **Redis** holds the city in shared memory so scenario switches load instantly. Required integrations aren't bolted on — they're the spine of the demo.

### 🧠 Technical difficulty
Real-time **LangGraph** orchestration (query → parallel **Linkup** research → Gemini risk-scoring → conditional impact / scenario nodes) streaming structured state over **AG-UI** into a **react-three-fiber** scene that materialises element-by-element with spring animations — plus a generative scene router, **Redis**-backed session memory, and strict Pydantic-validated agent output. High bar, fully wired.

### 💡 Originality
No team is building an **agent that picks and renders a 3D risk visualiser at runtime**. CityPulse targets a real, expensive market — emergency planners, insurers, urban developers and councils pay £50k+/year for slower, static, non-interactive risk tools — and replaces it with a live, queryable, agent-built map.

## How it works

```
                         ┌──────────────────────── LangGraph agent (FastAPI) ────────────────────────┐
  "London flooding" ──▶  │  query builder → Linkup research → Gemini risk-scoring → scene blueprint    │
                         │                                   └─ conditional: impact / scenario nodes   │
                         └───────────────────────────────────┬───────────────────────────────────────┘
                                                              │  A2UI envelopes over AG-UI (SSE)
                                                              ▼
                         ┌──────────────────────── Next.js + CopilotKit ─────────────────────────────┐
   3D scene materialises │  DynamicScene router reads blueprint.scene_type                            │
   zone-by-zone   ◀──────│   ├─ "city_grid"     → CityGridScene   (isometric grid of risk zones)      │
                         │   └─ "network_graph" → NetworkGraphScene (nodes + connections)             │
   click → A2UI card     │  A2UI renderer paints generated detail / impact surfaces                   │
                         └────────────────────────────────────────────────────────────────────────────┘
                                  ▲  Redis caches city state → scenario switches load instantly
```

**The scene blueprint** the agent emits (and the frontend renders):

```jsonc
{
  "scene_type": "city_grid",          // or "network_graph" — the agent decides
  "title": "London — Flooding Risk",
  "camera_preset": "isometric",
  "nodes": [
    { "id": "node_1", "label": "Canary Wharf", "type": "hub",
      "risk_score": 0.85, "position": { "x": -4, "y": 0, "z": -2 }, "size": 1.2 }
  ],
  "connections": [ { "from": "node_1", "to": "node_2", "risk_score": 0.3 } ]
}
```

Risk colour scale (shared across every scene):

| Score | Colour | Meaning |
|---|---|---|
| 0.0–0.3 | 🟢 `#4ade80` | Safe |
| 0.3–0.6 | 🟠 `#fb923c` | Moderate |
| 0.6–0.8 | 🔴 `#f87171` | High |
| 0.8–1.0 | 🔴 `#dc2626` | Critical (pulsing) |

## Quick start

```bash
# 1. Clone
git clone https://github.com/HeinHtet-Phyo/citypulse
cd citypulse

# 2. Install (Node app + Python agent)
pnpm install

# 3. Configure keys
cp .env.example .env
#   GEMINI_API_KEY=...   (https://aistudio.google.com/apikey)
#   LINKUP_API_KEY=...   (https://www.linkup.so)
#   REDIS_URL=... / REDIS_TOKEN=...   (https://upstash.com — free tier)

# 4. Run web app + agent together
pnpm dev
```

Open `http://localhost:3000`, type **`London — flooding risk`**, and watch the city build itself.

> **Backup for judging:** `OFFLINE=1 pnpm dev` paints a canned A2UI surface with no network call — your insurance if the venue Wi-Fi flakes mid-demo.
